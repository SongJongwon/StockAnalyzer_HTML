1. 수집하는 개인정보 항목

  1. 회원가입 시 (필수)
    • 이메일 주소
    • 비밀번호 (bcrypt 해시로 저장 — 평문 미보관)
    • 표시 이름
    • 휴대폰 번호 (선택, 입력 시)
  2. 유료 결제 시 (필수)
    • 결제 수단 정보 — PG사가 직접 수집·보관하며 회사는 거래 식별자만 저장 (자세한 항목은 PG사 약관 참고)
    • 결제 내역 (금액·플랜·일시)
  3. 서비스 이용 중 자동 수집
    • 접속 IP, 접속 일시, 브라우저·기기 정보 (보안 로그)
    • AI 분석·투자자 패널 호출 로그 (쿼터 관리 + 운영 통계)
    • 로그인 실패 카운터 (잠금 정책)
  4. BYOK 등록 시 (선택)
    • 본인 AI 제공자의 API 키 — 현재 브라우저에 한해 저장, Phase 4 에서 서버 측 AES-256 암호화 저장으로 전환 예정
  5. 고객 문의 시 (선택)
    • 이메일 주소, 문의 내용

⏸️ 마케팅 수신 동의 항목·이벤트 응모 항목 등은 추후 도입 시 본 항에 추가합니다.

2. 수집·이용 목적

  1. 회원 식별·본인 확인 — 로그인, 비밀번호 재설정, 부정 가입 방지
  2. 서비스 제공 — 차트·기술적 분석·AI 분석·투자자 패널 토론 등 핵심 기능 운영
  3. 유료 서비스 이용 관리 — 결제, 환불, 구독 갱신·해지, 쿼터 관리
  4. 고객 지원 — 문의 응답, 장애 안내, 공지 사항 발송
  5. 운영 통계·서비스 개선 — 사용량 통계, 페이지 사용 분석, 보안 모니터링 (개인 식별 불가 형태로 가공)
  6. 법령상 의무 이행 — 「전자상거래법」, 「전자금융거래법」 등에 따른 거래 기록 보관

3. 보유·이용 기간

회사는 수집한 개인정보를 회원이 서비스를 이용하는 기간 동안 보관하며, 회원 탈퇴 또는 보유 목적 달성 시 지체 없이 파기합니다. 단, 관련 법령에 따라 다음의 정보는 명시된 기간 동안 별도 보관합니다.

  • 계약·청약철회 기록: 5년 (전자상거래법 제6조 제3항)
  • 대금 결제·재화 공급 기록: 5년 (전자상거래법)
  • 소비자 불만·분쟁 처리 기록: 3년 (전자상거래법)
  • 웹사이트 방문 기록 (접속 로그·IP): 3개월 (통신비밀보호법)
  • 부정 이용 기록: 1년 (회사 자체 정책 — 관련 법령 우선)

4. 제3자 제공

회사는 회원의 개인정보를 본 방침에 명시한 목적과 다른 용도로 사용하거나 제3자에게 제공하지 않습니다. 다만 다음 각 호에 해당하는 경우 예외로 합니다.

  1. 회원이 사전에 동의한 경우
  2. 법령에 의해 요구되거나 수사 기관의 영장 등 적법 절차에 의한 요청이 있는 경우
  3. 통계 작성, 학술 연구 또는 시장 조사를 위해 개인 식별 불가 형태로 가공하여 제공하는 경우

5. 처리 위탁

회사는 원활한 서비스 제공을 위해 다음 업무를 외부에 위탁하고 있습니다. 위탁 시 「개인정보 보호법」 제26조에 따라 위탁 계약서에 개인정보 보호 의무·책임을 명시합니다.

6. 정보주체의 권리·행사 방법

회원은 언제든지 다음의 권리를 행사할 수 있습니다.

  1. 개인정보 열람·정정·삭제 요청 — 마이페이지에서 일부 항목 직접 수정 가능, 그 외에는 고객지원으로 요청
  2. 처리 정지 요청 — 서비스 이용에 필수적이지 않은 항목에 한해
  3. 회원 탈퇴 (개인정보 삭제) — Phase 3 완료 후 마이페이지 셀프 탈퇴 도입 예정. 현재는 고객지원 이메일 요청.
  4. 동의 철회 — 선택 항목에 대해 언제든 동의 철회 가능

법정대리인이 만 14세 미만 아동의 개인정보에 관한 권리를 행사하는 경우, 회사는 별도의 본인 확인 절차를 거칩니다.

7. 개인정보의 안전성 확보 조치

회사는 「개인정보 보호법」 제29조 및 같은 법 시행령 제30조에 따라 다음의 조치를 시행합니다.

  1. 관리적 조치 — 운영자 1인 단독 관리, 마스터 계정 2FA(TOTP) 적용
  2. 기술적 조치
    • 비밀번호 bcrypt cost 12 해싱 (평문 미보관)
    • JWT 기반 인증 + Refresh Token 분리
    • HTTPS 전 구간 적용
    • 로그인 실패 횟수 기반 자동 잠금
    • BYOK API 키 서버 측 AES-256 암호화 — Phase 4 도입 예정
  3. 물리적 조치 — Supabase·Render 등 위탁 처리자의 데이터센터 보안 정책 준수
  4. 접근 통제 — 마스터 권한 외 운영자 추가 시 IP 화이트리스트 적용 예정 (Phase 6)

8. 자동 수집 도구 (쿠키·localStorage)

  1. 사용 도구
    • localStorage — 로그인 토큰(access·refresh), 테마 설정(sa_theme), BYOK API 키 (Phase 4 전환 전까지)
    • HTTP 헤더의 Authorization Bearer 토큰
    • 현재 광고·마케팅 추적 쿠키는 사용하지 않습니다.
  2. 관리 방법 — 브라우저의 개발자 도구 또는 설정에서 localStorage·쿠키를 직접 삭제할 수 있으며, 삭제 시 자동 로그아웃됩니다.
  3. 거부 영향 — localStorage 사용을 거부할 경우 로그인 상태 유지·테마 설정 등 일부 기능이 제한됩니다.
